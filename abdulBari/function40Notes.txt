(2)-> isinstance(object, classinfo)
this function checks if object belongs to given class. 
In other words, is object an instance of given class
if yes, it returns true, else returns false


(3)-> hasattr(object, attribute)
to check if object/class has given attribute/method 

text = "Hello"
print(hasattr(text, 'lower')) #true
print(hasattr(text, 'search')) #false

str class has attribute/function called lower

We can check above for list, dict, set classes also


(4)-> getattr(object, attribute, default)
if a class or module has some attribute/method/function, it will give you reference to that attribute/method/function

(5)-> id()
calling it will give you unique number/id corresponding to that object

x = 10
y = 10
print(id(x))  
print(id(y))

ids will be same because both x and y are referring to same object 10


l1 = [1,2,3,4,5]
l2 = [1,2,3,4,5]
print(id(l1))  
print(id(l2))

why id of l1 and l2 are different?
Lists (l1, l2) are mutable and separately created

When you do:

l1 = [1,2,3,4,5]
l2 = [1,2,3,4,5]
print(id(l1))
print(id(l2))

Even though l1 and l2 contain the same elements,
Python creates two different list objects in memory.
